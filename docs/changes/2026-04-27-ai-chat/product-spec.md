## 背景与目标
- **业务背景**：用户需要一款轻量级、可自行配置的 AI 对话工具，能够自由接入不同的 AI 大模型服务，满足日常问答、编程辅助、文本处理等场景。
- **当前问题**：现有 AI 聊天产品通常绑定特定厂商的模型服务，无法自由切换模型提供商或自定义 API 地址；部分用户有私有化部署或使用国内模型的需求。
- **成功标准**：用户能够完成基本的对话交互，且可以自由配置任意兼容 OpenAI 协议的大模型服务的 URL、API Key 和模型 ID。

## 用户与场景
- **目标用户**：开发者、技术爱好者、日常 AI 工具使用者
- **典型场景**：用户在浏览器中打开 AI Chat，左侧展示历史会话列表，点击或新建会话后在右侧进行连续对话；用户可在设置中配置自己偏好的 AI 模型服务端点、密钥和模型名称。

## 用户故事
- **US-001**：作为用户，我希望在左侧看到所有历史会话列表，从而快速切换和回顾之前的对话。
- **US-002**：作为用户，我希望在右侧主区域进行实时对话，从而获得 AI 的即时回答。
- **US-003**：作为用户，我希望在设置页面配置自定义的 AI 模型 URL、API Key 和模型 ID，从而使用自己偏好的模型服务。
- **US-004**：作为用户，我希望新建一个空白会话，从而开始全新的对话主题。
- **US-005**：作为用户，我希望删除或重命名已有会话，从而更好地管理我的对话记录。

## 范围
### 本次要做
- **FP-001**：会话列表管理（新建、切换、删除、重命名会话）
- **FP-002**：对话交互区域（消息发送、AI 流式回复展示、消息历史加载）
- **FP-003**：模型配置管理（自定义 API URL、API Key、模型 ID 的配置与持久化存储）
- **FP-004**：后端 API 代理服务（转发前端请求至 AI 模型服务，处理流式响应）
- **FP-005**：会话数据的持久化存储（SQLite 数据库存储）

### 本次不做
- 用户登录与多用户体系
- 对话内容搜索
- 文件上传与多模态交互
- 对话分享功能
- 移动端适配（本次仅支持桌面端）

## 业务规则
- **BR-001**：API URL 必须为有效的 HTTPS/HTTP 地址，API Key 不能为空字符串，模型 ID 不能为空。
- **BR-002**：会话按最后活跃时间倒序排列，最新会话展示在最上方。
- **BR-003**：新消息发送时，需将当前会话的完整消息历史一并发送至模型服务，以保证上下文连贯。
- **BR-004**：模型配置保存后，后续所有对话请求均使用最新配置；配置变更不影响已有会话的展示。
- **BR-005**：后端流式响应需实时推送给前端，前端逐块渲染 AI 回复内容。

## 验收标准
- [ ] **AC-001**：用户打开应用后，左侧展示会话列表（空状态时显示提示文案），右侧展示欢迎界面或当前选中会话。
- [ ] **AC-002**：用户点击"新建会话"，左侧新增一个会话条目并自动选中，右侧清空为空白对话状态。
- [ ] **AC-003**：用户在输入框中输入文本并发送，消息出现在对话区域，AI 逐块返回回复内容并实时渲染。
- [ ] **AC-004**：用户在设置页面填入自定义 API URL、API Key 和模型 ID 并保存，重启应用后配置仍然有效。
- [ ] **AC-005**：用户删除会话后，会话从列表中移除；重命名会话后，列表中的会话名称立即更新。
- [ ] **AC-006**：切换会话后，历史消息完整加载到对话区域。

## 非功能性需求
- **NF-001**：页面初次加载时间不超过 3 秒（含后端启动）。
- **NF-002**：API Key 在后端存储时需加密处理，不得明文持久化。
- **NF-003**：支持主流现代浏览器最新两个版本（Chrome、Firefox、Edge、Safari）。
- **NF-004**：流式回复的首字到达时间不超过 2 秒（网络正常条件下）。

## 风险与依赖
- **依赖项**：Node.js 运行环境、React 前端框架、AI 模型服务 API 可用性
- **风险项**：不同 AI 模型服务的 API 协议可能存在差异，部分兼容性需适配
- **应对建议**：优先适配 OpenAI 兼容协议，其他协议通过适配层转换
