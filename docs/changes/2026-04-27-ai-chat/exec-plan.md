## 目标与完成定义
- **目标**：完成 AI Chat V1.0 的开发交付，支持会话管理、流式对话、自定义模型配置。
- **完成定义**：
  - [ ] 所有前后端功能开发完毕，前后端联调通过
  - [ ] 核心验收标准（AC-001 ~ AC-006）全部通过
  - [ ] API Key 加密存储功能验证通过
  - [ ] 项目可在本地一键启动并正常使用

## 背景与范围
- **当前问题**：无现成的可自定义模型接入的轻量级 AI 聊天工具。
- **推进原因**：满足个性化 AI 对话工具的日常使用需求。
- **本次范围**：会话 CRUD、流式对话交互、模型配置管理、数据持久化。
- **非本次范围**：用户认证、多模态交互、移动端适配、对话分享。

## 前置条件
- Node.js 18+ 运行环境已安装（SQLite 无需额外安装，由 better-sqlite3 内嵌处理）
- 选择一个 AI 模型服务的可用 API（如 OpenAI、兼容服务等）
- 确定前端 UI 组件库选型
- 确定 API Key 加密环境变量名称

## 阶段拆解

### 阶段一：项目初始化与基础搭建
- **目标**：搭建前后端项目骨架，建立目录结构和基础配置。
- **执行项**：
  - 初始化后端 Node.js 项目（Express + 基础中间件）
  - 初始化前端 React 项目（Vite + 基础路由）
  - 建立项目目录结构（路由、控制器、数据存储等目录）
  - 配置 ESLint、Prettier 等开发工具
- **产出**：可运行的空项目骨架，前后端能正常启动

### 阶段二：后端核心功能开发
- **TP-001**（关联 DS-001 / DS-004）：实现会话 CRUD API 和 SQLite 持久化
  - SQLite 数据库初始化（建库建表：conversations、messages、settings）
  - Express 路由和控制器实现
  - 使用 `better-sqlite3` 驱动封装数据访问层
- **TP-002**（关联 DS-002）：实现消息 API 和 AI 模型流式代理
  - 消息数据模型定义
  - `POST /api/conversations/:id/messages` 接口实现
  - SSE 流式转发 OpenAI 兼容 API 的响应
  - 流结束后自动持久化 AI 完整回复
- **TP-003**（关联 DS-003）：实现模型配置管理 API 和加密存储
  - 配置读写 API
  - AES-256 加密/解密工具模块
  - 环境变量读取加密密钥
- 预期工时：待确认

### 阶段三：前端核心功能开发
- **TP-004**（关联 DS-001 / US-001 / US-005）：实现会话列表 Sidebar 组件
  - 会话列表展示（空状态、加载态、列表渲染）
  - 新建、删除、重命名操作
  - 当前会话高亮状态管理
- **TP-005**（关联 DS-002 / US-002）：实现对话交互区 ChatArea 组件
  - 消息列表展示（用户消息 + AI 回复）
  - SSE 流式渲染（逐块追加内容）
  - 输入框与发送按钮
  - 加载历史消息（会话切换时）
- **TP-006**（关联 DS-003 / US-003）：实现模型配置设置页面
  - API URL 输入框（带校验）
  - API Key 输入框（密码模式）
  - 保存与加载配置
- 预期工时：待确认

## 风险与依赖
- **依赖项**：AI 模型 API 的可用性；Node.js 18+ 运行环境；前端 UI 组件库选型确定
- **风险项**：
  - 不同 AI 服务商对 SSE 的实现差异可能导致流式解析问题 — 建议优先适配 OpenAI 官方格式，非标格式通过适配层处理
  - API Key 加密若环境变量未配置可能导致服务启动失败 — 要求启动时检查环境变量，缺失时给出明确提示
- **当前阻塞**：无

## 验证与验收
- **验证方式**：开发阶段自测 → 阶段四联调 → 阶段五完整回归
- **验收标准**：
  - [ ] 全部验收标准 AC-001 ~ AC-006 通过
  - [ ] 流式对话首字到达时间 < 2s
  - [ ] API Key 加密存储验证通过
