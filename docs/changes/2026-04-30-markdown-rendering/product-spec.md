## 背景与目标
- **业务背景**：AI Chat V1.2 已支持基础对话、系统提示词、思考模式和天气查询 Agent。目前 AI 回复以纯文本形式展示，但 AI 模型回复内容普遍包含 Markdown 格式（代码块、表格、列表等），纯文本展示导致可读性差。
- **当前问题**：
  1. AI 回复中的代码块以原始文本展示，无语法高亮，难以阅读和调试。
  2. Markdown 表格、列表、标题等结构在纯文本中混成一团，失去层次感。
  3. 用户需要手动选中代码再复制，操作繁琐。
  4. 链接不可点击，需要手动复制到浏览器打开。
- **成功标准**：AI 回复中的 Markdown 内容正确渲染为格式化文本；代码块具备语法高亮和一键复制能力；表格、列表、链接等常见 Markdown 元素正常展示。

## 用户与场景
- **目标用户**：所有 AI Chat 用户（开发者为主）
- **典型场景**：
  - 用户询问"用 Python 写一个排序算法"，AI 回复包含代码块，展示为带语法高亮的代码区域。
  - 用户让 AI 生成一个对比表格，AI 回复中的 Markdown 表格渲染为结构化表格。
  - 用户看到 AI 回复中的代码块，点击右上角复制按钮一键复制代码。

## 用户故事
- **US-014**：作为用户，我希望 AI 回复中的 Markdown 格式（标题、列表、粗体、斜体、链接等）正确渲染，从而获得结构清晰、易于阅读的回答。
- **US-015**：作为用户，我希望代码块具有语法高亮，从而根据编程语言关键字颜色快速定位代码逻辑。
- **US-016**：作为用户，我希望代码块右上角有一键复制按钮，从而无需手动选择代码片段。
- **US-017**：作为用户，我希望 Markdown 表格渲染为表格形式，从而直观对比各列数据。
- **US-018**：作为用户，我希望 AI 回复中的链接可以直接点击跳转，从而无需手动复制 URL。
- **US-019**：作为用户，我期望行内代码有区别于普通文本的视觉样式，从而快速识别代码片段。

## 范围

### 本次要做
- **FP-013**：Markdown 渲染引擎集成 — 引入 react-markdown，将 AI 回复内容渲染为格式化 Markdown。
- **FP-014**：代码语法高亮 — 集成 highlight.js 或 shiki，支持常见编程语言的语法着色。
- **FP-015**：代码块复制按钮 — 每个代码块右上角显示复制按钮，点击复制源码到剪贴板。
- **FP-016**：代码块语言标签 — 在代码块左上角或右上角显示语言名称（如 `python`、`javascript`）。
- **FP-017**：表格渲染 — Markdown 表格渲染为 HTML `<table>`，附带基础样式。
- **FP-018**：链接安全处理 — 链接在新标签页打开，添加 `rel="noopener noreferrer"`。
- **FP-019**：行内代码样式 — `<code>` 标签应用独立背景色和等宽字体。
- **FP-020**：XSS 安全防护 — Markdown 渲染过程中过滤危险 HTML 标签和属性。

### 本次不做
- 用户输入侧的 Markdown 预览或编辑器（用户输入保持纯文本，不做渲染）。
- Markdown 数学公式（LaTeX）渲染。
- 流程图 / 时序图 / Mermaid 图表渲染。
- 自定义代码主题切换（仅提供一套默认主题，本次不做主题选择器）。
- 图片渲染（AI 回复中的图片 URL 保持为链接文本，不自动加载展示）。
- 任务列表（checkbox）的交互式勾选。
- 消息内容的编辑后重新渲染。

## 业务规则
- **BR-019**：Markdown 渲染仅应用于 AI（assistant）角色的消息内容，用户消息保持纯文本。
- **BR-020**：所有 Markdown 渲染输出必须经过 XSS 过滤，禁止执行 HTML `<script>`、`<iframe>`、`<object>` 等标签及 `on*` 事件属性。
- **BR-021**：代码块若指定了语言标识，需在代码块顶部或右上角显示语言标签；未指定语言时，仅展示代码块和复制按钮，不显示语言标签。
- **BR-022**：所有链接必须在浏览器新标签页打开，并附带 `target="_blank"` 和 `rel="noopener noreferrer"` 属性。
- **BR-023**：复制按钮复制的必须是源码文本（原始 Markdown 代码块内容），而非渲染后的 HTML。
- **BR-024**：流式输出过程中，内容增量逐块追加渲染，不应出现闪烁或未格式化文本闪烁后重新格式化的现象。
- **BR-025**：渲染组件必须能正确处理空内容、纯文本（无 Markdown 语法）和非法 Markdown 语法，不崩溃。

## 验收标准
- [ ] **AC-026**：AI 回复包含 `# 标题`、`**粗体**`、`*列表*` 等 Markdown 语法时，渲染为对应的格式化样式。
- [ ] **AC-027**：AI 回复包含代码块（\`\`\`python），代码块带有所述语言的语法高亮和语言标签。
- [ ] **AC-028**：点击代码块右上角复制按钮后，剪贴板内容为代码源码（不含语言标签、行号等额外内容）。
- [ ] **AC-029**：AI 回复中的 Markdown 表格渲染为带边框和交替行背景色的 HTML 表格。
- [ ] **AC-030**：AI 回复中的 Markdown 链接在外侧新标签页打开，且控制台检查无安全警告。
- [ ] **AC-031**：行内代码（反引号包裹）显示为带浅灰背景和等宽字体的独立样式。
- [ ] **AC-032**：AI 回复包含 `<script>alert('xss')</script>` 等危险内容时，不被执行（仅显示为文本或过滤）。
- [ ] **AC-033**：流式输出过程中，内容逐块追加渲染，无闪烁或"先纯文本后格式化"的视觉跳跃。
- [ ] **AC-034**：加载已有历史消息（之前纯文本格式），渲染正常不报错。
- [ ] **AC-035**：未指定语言的代码块渲染为普通代码块，显示复制按钮，无语言标签。
- [ ] **AC-036**：AI 回复包含纯文本（无任何 Markdown 语法）时，渲染样式与 V1.2 一致，无明显视觉变化。

## 非功能性需求
- **NF-011**：Markdown 相关依赖包的总 gzip 体积增量不超过 80KB（构建后）。
- **NF-012**：首次渲染性能：含 5 个代码块 + 1 个表格的典型消息，渲染时间不超过 100ms。
- **NF-013**：语法高亮覆盖至少 20 种常见编程语言（JavaScript、TypeScript、Python、Go、Rust、Java、C++、Ruby、Shell、SQL、HTML、CSS、JSON、YAML、Markdown 等）。
- **NF-014**：复制操作使用 `navigator.clipboard.writeText()` API，回退到 `document.execCommand('copy')`。
- **NF-015**：代码块支持水平滚动（超长行不换行截断），表格支持响应式布局。

## 风险与依赖
- **依赖项**：
  - `react-markdown`（或同类 Markdown 渲染库）
  - `rehype-highlight` / `highlight.js` / `shiki`（代码语法高亮）
  - 以上库需与 React 18 兼容。
- **风险项**：
  - 第三方 Markdown 库引入可能增加包体积，影响页面加载速度。
  - 流式渲染中逐块解析 Markdown 可能产生中间态的不完整格式渲染（如块还未闭合时的样式异常）。
  - XSS 风险：AI 模型可能输出含恶意 HTML 的内容，需严格过滤。
- **应对建议**：
  - 使用 `react-markdown`（tree-shakable） + 选择性引入 highlight.js 语言包，控制体积。
  - 流式渲染采用"内容稳定后再渲染"策略，每个消息块追加时将已累积的完整内容重新渲染，避免半截语法问题。
  - 使用 `rehype-sanitize` 或类似方案过滤危险 HTML，禁止危险标签和属性。

## 待确认事项
- 使用 `react-markdown` 还是 `marked` + `dangerouslySetInnerHTML` 方案？前者更安全（默认不解析 HTML），推荐 react-markdown。
- 语法高亮使用 highlight.js（轻量、生态好）还是 shiki（VS Code 风格、更美观但更重）？推荐 highlight.js。
- 代码主题选择哪个？（GitHub 风格 / VS Code Dark / 自定义）建议浅色模式用 GitHub 风格。
- 复制按钮的 Toast 反馈样式？（"Copied!" 提示显示位置和时长）
