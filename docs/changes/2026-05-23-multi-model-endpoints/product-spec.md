## 背景与目标
- **业务背景**：当前 Mint·清言 仅支持配置一组模型端点（API URL + API Key + Model ID），用户若需要在不同模型提供商之间切换（如 OpenAI GPT-4o ↔ Anthropic Claude ↔ 本地部署的 Ollama 模型），必须手动进入设置页重新填写所有字段，操作繁琐、易出错，且无法快速对比不同模型的回答质量。
- **当前问题**：
  1. 仅支持单一端点配置，切换模型需进入设置页重新填写 URL、Key、Model ID 三项，用户体验差。
  2. API Key 输入后仅显示脱敏占位符，切换回来时无法判断之前配置的是哪个 Key，容易误覆盖。
  3. 没有模型标识，对话中不知道自己当前用的是哪个模型。
  4. 设置页的表单校验假设"必须填 API Key"，但本地模型（如 Ollama）通常无需 Key，导致无法保存。
- **成功标准**：
  - 用户可保存多组模型端点配置（每组含名称、API URL、API Key、Model ID），并可在对话界面一键切换。
  - 切换模型后新对话立即生效，当前对话不受影响。
  - 从旧版单配置平滑迁移，现有配置不丢失。

## 用户与场景
- **目标用户**：所有 Mint·清言 用户（特别是同时使用多个模型提供商的用户）
- **典型场景**：
  - 用户配置了三组端点——"OpenAI GPT-4o"、"Claude Opus 4.7（代理）"、"本地 Ollama Qwen3"——在对话中先问 GPT-4o 一个编程问题，不满意后点击模型选择器切换到 Claude Opus 重新提问，全程无需进入设置页。
  - 用户在设置页新增一组端点配置，填写名称、URL、Key、Model ID，保存后该配置出现在模型选择器中。
  - 用户删除了一个不再使用的端点配置，该配置从列表和选择器中移除，但不影响使用该配置的历史对话。

## 用户故事
- **US-036**：作为用户，我希望保存多组模型端点配置（名称 + API URL + API Key + Model ID），从而无需每次切换模型时重新输入。
- **US-037**：作为用户，我希望在对话界面通过下拉选择器快速切换当前使用的模型端点，从而一键在不同模型间切换。
- **US-038**：作为用户，我希望在设置页管理端点配置（新增、编辑、删除），且删除端点时有确认提示，避免误操作。
- **US-039**：作为用户，我希望从旧版单配置自动迁移到多配置模式，已有的 API URL、Key、Model ID 不丢失。
- **US-040**：作为用户，我希望 API Key 为可选项（本地模型通常不需要 Key），保存时不再强制要求填写。

## 范围

### 本次要做
- **FP-037**：多端点配置存储 — 数据库新增 `model_endpoints` 表，支持存储多组端点配置（名称、API URL、API Key、Model ID、排序），其中一组标记为当前激活。
- **FP-038**：端点管理 API — 新增端点 CRUD 接口（GET 列表、POST 新增、PUT 更新、DELETE 删除、PUT 激活），API Key 返回时脱敏。
- **FP-039**：设置页端点管理 UI — 在设置面板新增"模型端点"Tab，支持列表展示、新增/编辑弹窗、删除（含确认）、设为当前端点。
- **FP-040**：对话界面模型选择器 — 在 ChatArea 顶部新增模型选择下拉框，显示已配置的端点名称列表，点击即可切换，当前选中项高亮。
- **FP-041**：旧版数据迁移 — 首次启动时自动将 `settings` 表中的旧配置迁移到 `model_endpoints` 表，标记为默认端点。
- **FP-042**：API Key 可选化 — 保存端点时 API Key 不再强制必填，允许空值以支持无需认证的本地模型。

### 本次不做
- 端点配置的导入/导出功能。
- 基于对话上下文自动推荐模型切换。
- 端点可用性健康检查（ping 测试）。
- 不同对话绑定不同端点（全局激活端点对所有对话生效，历史对话加载时使用当时的消息内容）。
- 端点配置的云端同步。

## 业务规则
- **BR-054**：端点名称不能为空，不能与已有端点重名（同一用户范围内），最大 50 字符。
- **BR-055**：API URL 必须为有效 URL 格式，不可为空。
- **BR-056**：Model ID 不可为空。
- **BR-057**：API Key 允许为空（支持无需认证的本地模型）。
- **BR-058**：至少保留一个端点配置，删除最后一个端点时拒绝操作并提示"至少保留一个端点"。
- **BR-059**：切换激活端点后，新对话使用新端点；当前对话不受影响（继续使用发起对话时的端点）。
- **BR-060**：首次启动时若 `model_endpoints` 表为空，自动从旧 `settings` 表迁移数据，迁移后旧配置保留不删除。
- **BR-061**：模型选择器显示在 ChatArea 顶部，显示当前激活端点的名称，点击展开下拉列表。
- **BR-062**：端点删除时弹出确认对话框，明确提示"删除后不可恢复"。

## 验收标准
- [ ] **AC-067**：用户可在设置页添加多组端点配置（名称、URL、Key、Model ID），保存后在列表中可见。
- [ ] **AC-068**：模型选择器显示所有已配置端点，点击切换后新发送的消息使用新端点，回复风格与所选模型一致。
- [ ] **AC-069**：编辑已有端点配置后，使用该端点的后续对话立即反映新配置。
- [ ] **AC-070**：删除非激活端点成功，删除激活端点后自动激活列表中的第一个端点。
- [ ] **AC-071**：删除最后一个端点时被拒绝，提示"至少保留一个端点"。
- [ ] **AC-072**：API Key 留空可保存成功，使用该端点发消息不报错（适用于本地模型）。
- [ ] **AC-073**：从旧版升级到新版后，原有端点配置自动迁移到新系统中，名称默认为"默认端点"。
- [ ] **AC-074**：API Key 在接口返回中始终脱敏（如 `sk-...****`），仅在 AI 调用时解密使用。

## 非功能性需求
- **NF-003**：端点切换操作响应时间 < 200ms（纯前端切换选中态，无网络请求）。
- **NF-004**：端点 CRUD 接口响应时间 < 500ms。
- **NF-005**：API Key 存储使用现有 AES-256-GCM 加密方案，不引入新的加密机制。

## 风险与依赖
- 依赖项：现有 AES-256-GCM 加密模块（`server/services/encryption.ts`）。
- 风险项：旧版数据迁移失败可能导致用户配置丢失——通过保留旧 `settings` 表数据不删除来降低风险；若加密 Key 变更导致旧 API Key 解密失败，迁移时标记该字段为空并提示用户重新输入。
