# 产品规格：图片生成对话化改造

## 背景与目标

### 背景

当前图片生成功能以独立表单形式呈现（`ImageGenerator` 组件），用户在"生图"视图中填写提示词、选择参数、点击生成、查看图片。这与文本对话的聊天形式严重不一致，用户在两种模式间切换时有明显的体验割裂感。

### 目标

将图片生成改造为与文本对话一致的聊天形式，让"生图"视图拥有独立的对话列表，用户在对话中以"发消息→收图片"的方式完成图片生成，保留完整对话历史。

## 用户与场景

| 用户 | 场景 |
|------|------|
| AI 绘画用户 | 在"生图"视图中创建对话，输入提示词生成图片，后续可追加修改要求或查看历史 |
| 设计师 | 通过多轮对话逐步调整图片风格和内容，每次迭代生成新图片 |

## 用户故事

| ID | 描述 | 优先级 |
|----|------|--------|
| US-001 | 作为用户，我可以在侧边栏切换到"生图"视图，看到独立于文本对话的图片对话列表 | P0 |
| US-002 | 作为用户，我可以在"生图"视图中创建、重命名、删除图片对话 | P0 |
| US-003 | 作为用户，我可以在图片对话中输入提示词并发送，AI 回复生成的图片，交互流程与文本对话一致 | P0 |
| US-004 | 作为用户，我可以在发送前配置图片参数（尺寸/质量/格式），这些控件在输入区附近可见 | P1 |
| US-005 | 作为用户，我可以在切换回旧对话时看到之前生成的所有图片，图片随消息一起渲染 | P0 |

## 范围

### 包含

- 侧边栏"生图"视图下显示**图片对话列表**（独立于文本对话）
- 图片对话采用与文本对话相同的**消息列表 + 输入框**布局
- 输入框区域附带**图片参数控件**（模型选择、尺寸、质量、格式）
- 消息列表中**渲染图片结果**（包括图片、优化后的提示词）
- 对话切换时**重新加载历史图片**
- 图片对话的创建/删除/重命名

### 不包含

- 图片编辑/修图/局部重绘（inpainting）
- 一次消息生成多张图片（保持 n=1，多个图片可放在同一回复中）
- 图片本地缓存/下载（图片 URL 保持外部引用）
- SSE 流式生成过程展示（图片生成是请求-响应模式，非流式）
- 图片对话的 Agent 路由、ReAct 工具调用、记忆机制

## 业务规则

| ID | 规则 | 说明 |
|----|------|------|
| BR-001 | 图片对话类型 | 使用 `conversations` 表，新增 `type='image'` 字段标识，与 `type='text'` 的文本对话共存 |
| BR-002 | 图片消息存储 | assistant 角色消息新增 `image_data` 字段存储图片结果 JSON（URL、revised_prompt 等） |
| BR-003 | 模型筛选 | 图片对话只能选择 `category='image'` 的模型端点，文本对话不可见图生图模型 |
| BR-004 | 参数持久化 | 每条消息的生成参数随消息存储，历史回看时参数不可编辑（仅展示） |

## 验收标准

| ID | 标准 | 关联 |
|----|------|------|
| AC-001 | 侧边栏"生图"视图切换后，对话列表只显示 type=image 的对话，顶部显示"新建"按钮 | US-001, US-002 |
| AC-002 | 在图片对话中发送消息，用户消息显示提示词，AI 回复显示图片 + 优化后提示词 | US-003 |
| AC-003 | 发送消息前可修改模型/尺寸/质量/格式，参数选择器位于输入框上方或附近 | US-004 |
| AC-004 | 切换对话后重新加载，历史图片正常显示 | US-005 |
| AC-005 | 文本对话视图中不可见图片对话，图片对话视图中不可见文本对话 | US-001 |

## 风险与依赖

| 风险 | 影响 | 缓解 |
|------|------|------|
| 图片 URL 可能过期 | 历史图片可能无法加载 | 前端 img 标签 onError 展示占位图 |
| 生产环境已有 text 对话数据 | 需确保新追加 type 列的迁移兼容旧数据 | type 列默认值设为 'text'，已有数据自动兼容 |
