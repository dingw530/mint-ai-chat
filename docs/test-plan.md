## 2. 测试策略

### 2.1 测试层级

| 层级 | 目标 | 工具/框架 | 责任方 |
|---|---|---|---|
| 单元测试 | 加密工具、数据访问层 | Vitest | 开发工程师 |
| API 集成测试 | 后端 API 接口全覆盖 | Vitest + fetch | 测试工程师 |
| 前端组件测试 | React 组件渲染与交互 | Vitest + Testing Library | 前端工程师 |
| E2E 测试 | 完整业务流程验证 | Playwright | 测试工程师 |

### 2.2 测试数据管理

- **数据库**：每次测试运行使用独立的测试数据库文件 `test-ai-chat.db`
- **清理策略**：`beforeAll` 清理旧测试库，`afterAll` 删除测试库文件及 WAL 文件
- **种子数据**：每个测试用例独立准备所需数据，避免测试间相互污染
- **环境隔离**：通过 `NODE_ENV=test` 环境变量隔离测试配置

### 2.3 测试环境

| 项目 | 配置 |
|---|---|
| Node.js | 18+ |
| 数据库 | SQLite（better-sqlite3 内嵌） |
| 加密密钥 | 测试环境自动生成随机密钥 |
| 端口 | 3099（测试专用，避免与开发端口冲突） |

### 2.4 测试执行命令

```bash
# 运行所有测试
npx vitest run

# 运行特定测试文件
npx vitest run server/__tests__/api.test.js
npx vitest run server/__tests__/encryption.test.js

# 带覆盖率报告
npx vitest run --coverage

# 监听模式（开发时使用）
npx vitest
```

## 5. 缺陷分级标准

| 级别 | 定义 | 响应时间 |
|---|---|---|
| P0 阻塞 | 核心功能无法使用（如无法发送消息、无法保存配置） | 立即修复 |
| P1 严重 | 功能无法按预期工作（如流式渲染中断、加密失效） | 24小时内 |
| P2 一般 | 功能可用但体验受影响（如错误提示不明确、列表顺序错误） | 下个迭代 |
| P3 建议 | 轻微 UI 问题或优化建议 | 待排期 |

## 7. 风险与应对

| 风险 | 影响 | 应对措施 |
|---|---|---|
| 第三方 AI 模型 API 不可用 | 流式消息测试无法通过 | 使用 Mock Server 模拟 SSE 响应 |
| 加密密钥环境变量未配置 | 加密功能失效 | 测试环境使用自动生成的随机密钥 |
| 数据库文件残留 | 测试结果不准确 | `beforeAll`/`afterAll` 清理测试数据库 |
| SSE 在 CI 环境中行为差异 | 流式测试不稳定 | 增加超时容忍、使用 Mock 替代真实 API 调用 |
